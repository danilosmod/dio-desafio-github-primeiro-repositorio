Criação de um token de acesso pessoal - GitHub

Você deve criar um token de acesso pessoal para usar no lugar de uma senha com a linha de comando ou com a API.

Nota: Se você usar a CLI do GitHub para autenticar no GitHub na linha de comando, poderá pular a geração de um token de acesso pessoal e autenticar por meio do navegador da web. Para obter mais informações sobre a autenticação com GitHub CLI, consulte gh auth login.

Os tokens de acesso pessoais (PATs) são uma alternativa ao uso de senhas para autenticação no GitHub ao usar a API do GitHub ou a linha de comando .

Se quiser usar uma PAT para acessar recursos de propriedade de uma organização que usa SAML SSO, você deve autorizar a PAT. Para obter mais informações, consulte " Sobre a autenticação com logon único SAML " e " Autorizando um token de acesso pessoal para uso com logon único SAML ".

Como medida de segurança, o GitHub remove automaticamente tokens de acesso pessoais que não são usados ​​há um ano. Para fornecer segurança adicional, é altamente recomendável adicionar uma expiração aos seus tokens de acesso pessoais.

Um token sem escopos atribuídos só pode acessar informações públicas. Para usar seu token para acessar repositórios na linha de comando, selecione repo. Para obter mais informações, consulte “ Escopos disponíveis ”.

Criação de um token

  1. Verifique seu endereço de e-mail , caso ainda não tenha sido verificado.
  2. No canto superior direito de qualquer página, clique na foto do seu perfil e em Configurações .

  3. Na barra lateral esquerda, clique em Configurações do desenvolvedor .

  4. Na barra lateral esquerda, clique em Tokens de acesso pessoais .

  5. Clique em Gerar novo token .

  6. Dê ao seu token um nome descritivo.

  7. Para dar ao seu token uma expiração, selecione o menu suspenso Expiração e clique em um padrão ou use o seletor de calendário.

  8. Selecione os escopos ou permissões que deseja conceder a este token. Para usar seu token para acessar repositórios na linha de comando, selecione repo .

  9. Clique em Gerar token .

  10. Para usar seu token para autenticação em uma organização que usa SAML SSO, autorize o token para uso com uma organização de logon único SAML .

Usando um token na linha de comando

Depois de ter um token, você pode inseri-lo em vez de sua senha ao executar operações Git sobre HTTPS.

Por exemplo, na linha de comando, você digitaria o seguinte:

$ git clone https://github.com/username/repo.git

Username: your_username

Password: your_token

Os tokens de acesso pessoal só podem ser usados ​​para operações HTTPS Git. Se o seu repositório usa um URL remoto SSH, você precisará mudar o remoto de SSH para HTTPS .

Se não for solicitado seu nome de usuário e senha, suas credenciais podem ser armazenadas em cache no computador. Você pode atualizar suas credenciais no Keychain para substituir sua senha antiga pelo token.

Em vez de inserir manualmente seu PAT para cada operação HTTPS Git, você pode armazenar em cache seu PAT com um cliente Git. Git irá armazenar temporariamente suas credenciais na memória até que um intervalo de expiração tenha passado. Você também pode armazenar o token em um arquivo de texto simples que o Git pode ler antes de cada solicitação. Para obter mais informações, consulte " Armazenando em cache suas credenciais do GitHub no Git ."
